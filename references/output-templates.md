# Output Templates

Use these templates to produce direct build artifacts, not long idea lists.

```markdown

- Name:
- One-line pitch:
- Target user:
- Core pain:
- Why this is shareable:

## MVP Features

- Feature 1:
- Feature 2:
- Feature 3:

## Tech Stack

- Frontend:
- Backend:
- Storage:
- AI layer:
- Deployment:

## Repo Structure

```text
repo-name/
  app/
  components/
  lib/
  public/
  README.md
```

## README Hook

[2-4 lines]

## Demo Plan

- Input:
- Flow:
- Output:
- GIF or screenshot moment:

## Launch Notes

- First post angle:
- First community angle:

```
